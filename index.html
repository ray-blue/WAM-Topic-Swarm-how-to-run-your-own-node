Talk & Workshop 1 Run your Bee node

In this talk, the Swarm team will explain how the Swarm network works, what part nodes play in it and how to spin up your own node.


Workshop 2 how to set up your Fluence node

Fluence provides an open-source, permissionless, decentralized platform and a development toolkit for building, hosting, and running peer-to-peer applications and protocols. The Fluence team will take you through the steps of setting up your own Fluence node.


Workshop 3 How to run a NYM node

Learn directly from the NYM team how to become a node operator in their network. NYM runs a multi-layered network called a mixnet. In each layer of the mixnet, mix nodes mix your internet traffic with that of other users, making communications private and hiding your metadata.


Workshop 4 Avado demo: running a Swarm data node

Avado provides specialized HW and SW that allows you to easily run and manage different network nodes. This workshop will demonstrate how to run a Swarm node on Avado.

Panel discussion; Hosting distributed networks - Relevance and Impact

Distributed networks were created with the purpose to remediate the shortcomings of centralised networks of Web2. In his talk, Denis Roio will unpack what distributed networks are and why they matter. The talk will be held via Twitter Spaces.


My focus though will be on how to run your Bee node and what its about. Anatol talked us through the swarm project and presented the demo.


Why would someone want a node?

Becoming a Bee client means you can be up and running in no time, it instantly exposes you to the ecosystem were you can begin to connect with peers all over the world and become apart of Swarm.Which is a global p2p network tasked with storing and distributing all of the worlds data onto Swarm.This new network has been built to be painless and user friendly. You can also join the swarm with no Ethereum backend and no Crypto onboarding.


Light Node(Full Node)

The Bee runs as a light node which is chain enabled. However, to fully engage in the swarm you need to set it to full-node. Being in full mode means you have access to the network and can easily upload and download files. But before you can do that you need to follow a few steps:

  • Fund your node’s wallet with BZZ.
  • Purchase a batch of stamps with your BZZ.
  • Wait for the batch to propagate into the network.
  • Upload your content, specify the batch id so that Bee can attach stamps to your chunks
  • Download your content using your contents hash.


Ultra light Node

Ultra light allows for the Bee to operate in a chain-less mode. This means you have access to the network but can not participate in most of the activities such as storing chunks, forwarding or uploads. The only activity you can participate in is downloading files.
The upside is that onboarding is easier,and you can somewhat still use the node. Furthermore you can enjoy the benefit of free riding.


Bee Dashboard

The Dashboard is still under development. However, the idea around the Bee Dashboard is to monitor your transactions of cheques as they accumulate,withdraw BZZ and many more features are under development and are coming soon.


What Hardware can be used?

Ultra light node A Raspberry Pi 4 and a Smartphone can be used. Light node(Full Node), Any computer that's connected to the internet can be used as well as cloud hosting and a Raspberry PI.

Disadvantages:

  • The Overlay verification process, A valid overlay address is required.
  • The amount of downloads are yet to be determined.
  • How much memory consumption is used and need, is still to be tested.
  • Bandwidth.
  • How?, and why do you get blocked, and for how long.

All you need to get started

Bee is packaged for MacOS and Ubuntu, Raspbian, Debian and CentOS based Linux distributions. If your system is not supported, please see the manual installation section for information on how to install Bee. To install Bee you will need to go through the following process.

  • Set up the external signer for Bee, Bee Clef. (Recommended)
  • Install Bee and set it up to run as a service.
  • Configure Bee.
  • Fund your node with XDAI and BZZ for the mainnet and gETH and gBZZ for our testnet.
  • Wait for your cheque book transactions to complete and batch store to update.
  • Check Bee is working.
